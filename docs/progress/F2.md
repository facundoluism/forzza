## HUMAN_REQUIRED

1. **Cuenta Supabase**: Crear cuenta en supabase.com (gratis) → New Project.
2. **Docker corriendo**: Abrir Docker Desktop y esperar "Engine running".
3. **Inicializar Supabase local**:
   ```bash
   cd C:\dev\forzza
   supabase login
   supabase start
   pnpm db:reset
   pnpm test:rls
   pnpm db:types
   ```
4. **Linkear al proyecto cloud** (opcional para dev local):
   ```bash
   supabase link --project-ref <ref-del-proyecto>
   ```
5. **Cargar claves en .env**: copiar NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY desde supabase.com → Settings → API.

## Notas técnicas

- `audit_log` y `processed_events` tienen REVOKE UPDATE/DELETE para ser append-only.
- El trigger `check_coach_package_price` usa `country_config.min_coach_price` en centavos.
- AR: piso = 500000 centavos = $5000 ARS. CL: piso = 2000000 = $20000 CLP.
- El trigger `update_billing_model_on_assignment` pasa a 'comision' al 4° alumno activo y NUNCA revierte.
- `exercise_library`: 30 ejercicios base. El script de importación del Excel irá en scripts/import-exercises.ts (Fase 2.5 o post-V1 cuando el Excel esté disponible).
- Chile está en country_config con `active = false`. Las rutas/UI de Chile se activan en V1.5.
