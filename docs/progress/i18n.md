# i18n completo ES/EN — App mobile (2026-06-15)

> Desvío de spec V1 aprobado por el owner (ver docs/open-questions.md). CLAUDE.md
> define V1 español-only; el owner pidió app bilingüe ES/EN con selector en el perfil
> y los 234 ejercicios en ambos idiomas. Proyecto multi-sesión.

## Estado de datos al inicio
- `exercise_library.name` = español ✅ · `name_en` = inglés ✅ (nombres ya bilingües)
- `exercise_library.description` = **solo inglés** ❌ (falta español)
- `primary_group`, `difficulty`, `primary_muscles`, `secondary_muscles`, `movement_pattern`, `equipment`, `tags` = **solo inglés** ❌
- App mobile: **sin infraestructura i18n**; perfil = placeholder sin selector.

## Plan por fases

### Fase 1 — Fundación + selector  ⏳ EN PROGRESO
- Librería i18n (`i18next` + `react-i18next` + `expo-localization`).
- Store de idioma (Zustand + persistencia AsyncStorage), default desde locale del device.
- Recursos `es`/`en` (JSON namespaced).
- **Selector de idioma en el perfil** (perfil reconstruido con tokens del design system).
- Migrar strings de las 5 tabs + chrome de los componentes de ejercicios a claves i18n.
- Nombre de ejercicio: mostrar `name` o `name_en` según idioma.

### Fase 2 — Resto de pantallas
- Migrar strings de: auth (login/signup/onboarding/forgot/reset), marketplace (index/[coachId]/checkout), session, routine (new/[id]), notifications, chat, upgrade, styleguide. Fan-out por grupos a subagentes.

### Fase 3 — Contenido de ejercicios bilingüe
- Migración DB: `description_es` + `description_en` (backfill `description_en` desde `description` actual; generar `description_es`).
- Mapas ES/EN para vocabulario controlado (group, difficulty, movement_pattern) y abierto (músculos, equipamiento) en un módulo de constantes.
- Wire de `ExerciseLibraryPicker` / `ExercisePreviewSheet` al idioma activo.
- Las 234 descripciones ES autogeneradas → **HUMAN_REVIEW** (terminología fitness, voseo).

### Fase 4 (a confirmar) — Web
- Replicar i18n en apps/web si el owner lo confirma.

## Criterio de cierre
Cada fase: typecheck + lint verdes; selector cambia idioma en caliente y persiste; sin strings hardcodeados en las pantallas migradas.
