## HUMAN_REQUIRED

1. **Supabase funcionando** (Fase 2): sin DB activa, el auth no funciona.
2. **Variables de entorno**: agregar al `.env` del web `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`. Para mobile: `EXPO_PUBLIC_SUPABASE_URL`, `EXPO_PUBLIC_SUPABASE_ANON_KEY`.
3. **Edge Function delete-account**: cuando DB esté activa, implementar la función que hace soft-delete del user, cancela subs y limpia datos.
4. **Resend (emails de auth)**: Supabase envía emails de verificación/recuperación con su SMTP interno en modo local. Para producción: configurar Resend como SMTP custom en Supabase Dashboard.
5. **Apple Sign In / Google Sign In**: activar en `packages/config/src/index.ts` `FEATURE_FLAGS` una vez que tengas las credenciales OAuth.

## Notas técnicas

- `parental_consent_at` se guarda en DB; la verificación server-side para checkout de coach (403) va en Fase 11.
- El supabase client mobile usa `expo-secure-store` para persistencia segura de la sesión.
- El supabase client web usa `@supabase/ssr` con cookies para SSR/RSC.
- Los colores en styles son de los tokens de packages/ui/src/tokens.ts (hardcoded temporalmente hasta Fase 4).
