## HUMAN_REQUIRED

1. **Completar `EAS_OWNER`** en `app.config.ts` con el nombre de usuario de Expo.com del dueño.
2. **App Store Connect** — Crear app con bundle `com.forzza.app`; configurar paid apps agreement y definir entidad legal (bloqueante para publicar).
3. **Google Play Console** — Crear app; completar Data Safety y formularios de contenido.
4. **Primer `eas build`** — Una vez activas las cuentas: `eas build --platform all --profile production`.
5. **TestFlight interno** — Subir build iOS al grupo de testers internos antes de open beta.

## Próximos pasos

1. Resolver entidad legal (el bloqueante más crítico para publicación en App Store).
2. Ejecutar `eas build --platform android --profile preview` (no requiere cuenta Play Console para generar APK).
3. Instalar el APK en dispositivo Android y ejecutar smoke test manual: signup → onboarding → crear rutina → iniciar entreno.
4. Una vez resuelto App Store Connect, ejecutar build iOS y distribuir por TestFlight a los primeros 5 testers.
